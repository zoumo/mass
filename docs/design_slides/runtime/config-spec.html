§1 规范版本

规范版本字段定义了配置文件所遵循的 OAR Runtime Specification 版本，确保消费方能够正确解析和验证配置。

字段 类型 约束 说明
oarVersion string REQUIRED 必须符合 SemVer v2.0.0 格式

版本控制特性

  • • 遵循 SemVer 语义化版本控制，主版本内保持兼容
  • • 消费方必须拒绝未知的主版本号，防止解析错误
  • • 向前兼容：新版本可读取旧版本配置
  • • 向后兼容：旧版本可读取新版本配置（主版本相同）

示例代码 

{
  "oarVersion": "0.1.0"
}

§2 元数据

元数据字段描述 agent 实例的身份信息，包括实例名称和可选的注解。这是实例级别的标识，而非类型定义。

字段 类型 约束 说明
metadata object REQUIRED Agent 实例身份信息容器
name string REQUIRED 实例名称，如 "auth-refactor-agent"
annotations map[string]string OPTIONAL 任意元数据，键使用反向域名表示法

设计理念

  • • 实例 vs 类型：name 是实例名（如 "auth-refactor-agent"），而非 agent 类型名（如 "claude"）
  • • OCI 一致性：与 OCI config.json 设计一致，描述具体容器实例而非镜像
  • • 注解规范：键使用反向域名表示法（如 org.openagents.task）

示例代码 

{
  "metadata": {
    "name": "auth-refactor-agent",
    "annotations": {
      "org.openagents.task": "backend-refactor",
      "org.openagents.owner": "zoumo"
    }
  }
}

§3 AgentRoot

AgentRoot 指定 agent 的工作目录根路径，类比 OCI 的 root 字段。关键设计：路径相对于 bundle 目录，而非绝对路径。

字段 类型 约束 说明
agentRoot object REQUIRED Agent 工作目录配置
path string REQUIRED 相对于 bundle 的路径，通常为 "workspace"

与 OCI root 的对比

OCI OAR
root.path — rootfs 相对路径 agentRoot.path — workspace 链接相对路径
containerd 通过 snapshotter 准备 rootfs agentd Workspace Manager 准备 workspace 并 symlink
runc 解析路径作为容器 / agent-shim 解析路径作为 agent cwd
rootfs 每容器隔离 workspace 在 Room 中共享

agent-shim 解析流程

  • 1 Join(bundle目录, agentRoot.path) → 得到绝对路径
  • 2 EvalSymlinks → 获取 canonical 路径
  • 3 用作 cmd.Dir 和 ACP session/new cwd

Bundle 目录结构 

/var/lib/agentd/bundles/session-abc123/
├── config.json                ← agentd writes
└── workspace -> /var/lib/agentd/workspaces/ws-def456/   ← agentd symlinks

agent-shim 解析 workspace → /var/lib/agentd/workspaces/ws-def456/

§4 ACP Agent

ACP Agent 是配置的核心部分，包含三个关键组件：systemPrompt（角色定义）、process（进程启动）、session（会话参数）。字段名显式包含 "acp"，表明仅支持 ACP 协议。

字段 类型 约束 说明
acpAgent object REQUIRED ACP agent 运行时配置
systemPrompt string OPTIONAL Agent 角色定义和能力约束
process object REQUIRED 进程启动配置
session object OPTIONAL ACP session/new 参数

SystemPrompt

Agent 的核心身份属性，定义角色和能力约束。在 Room 场景中，agent 先启动就绪，等待编排器通过 ARI session/prompt 发送任务。

  • • 来源：runtimeClass 配置 + 编排器请求
  • • 传递时机：ACP session/new
  • • 层级：与 process、session 平级

Process

指定如何启动 ACP agent 进程。运行时使用这些字段进行 fork/exec。

字段 类型 约束 说明
command string REQUIRED 可执行文件，类似 OCI process.args[0]
args []string OPTIONAL 命令行参数
env []string OPTIONAL 环境变量覆盖，格式 KEY=VALUE
环境变量合并策略
  • 1 agentd 合并 runtimeClass 配置（如 ANTHROPIC_API_KEY）+ 编排器请求（如 GITHUB_TOKEN）
  • 2 运行时以父进程环境为基础，config.json 变量覆盖其上
  • 3 确保 agent 拥有完整系统环境（PATH、HOME 等），config.json 优先级更高

Session

指定 ACP session/new 请求参数。完成 initialize 握手后，运行时发送 session/new 创建会话。

字段 类型 约束 说明
mcpServers []McpServer OPTIONAL MCP 服务列表，默认 []
McpServer 结构
字段 类型 约束 说明
type string REQUIRED 传输类型："http" 或 "sse"
url string REQUIRED MCP 服务 URL

Process 示例 

{
  "acpAgent": {
    "process": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/claude-code-acp"],
      "env": [
        "ANTHROPIC_API_KEY=sk-ant-xxx",
        "GITHUB_TOKEN=ghp_xxx"
      ]
    }
  }
}

Session 示例 

{
  "acpAgent": {
    "session": {
      "mcpServers": [
        {
          "type": "http",
          "url": "http://localhost:3000/mcp"
        }
      ]
    }
  }
}

§5 Permissions

Permissions 字段指定 agent-shim 处理 fs/* 和 terminal/* 请求时的权限策略，控制 agent 对文件系统和终端的访问权限。

字段 类型 约束 说明
permissions string OPTIONAL 权限策略，默认 "approve-all"

权限策略取值

值 行为
"approve-all" 所有 fs/terminal 操作自动批准
"approve-reads" 只读操作批准，写操作返回 deny
"deny-all" 所有操作返回 deny

关键特性

  • • 策略在 session 创建时确定，运行时不可更改
  • • 影响 fs/* 和 terminal/*两类请求
  • • 默认值 "approve-all" 允许完全访问

示例代码 

{
  "permissions": "approve-all"
}

§6 完整示例

三种典型场景的完整配置：带角色设定的 Agent、空闲 Agent（Room 成员）、最小配置。

示例 1 带角色设定的 Agent

runtimeClass "claude" 解析后的完整配置，启动后等待编排器通过 ARI session/prompt 发送任务。 

{
  "oarVersion": "0.1.0",

  "metadata": {
    "name": "auth-refactor",
    "annotations": {
      "org.openagents.task": "PROJ-1234"
    }
  },

  "agentRoot": {
    "path": "workspace"
  },

  "acpAgent": {
    "systemPrompt": "你是专注于安全和认证系统的高级后端工程师。请遵循项目的编码规范。",
    "process": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/claude-code-acp"],
      "env": [
        "ANTHROPIC_API_KEY=sk-ant-xxx",
        "GITHUB_TOKEN=ghp_xxx"
      ]
    },
    "session": {
      "mcpServers": [
        {
          "type": "http",
          "url": "http://localhost:3000/mcp"
        }
      ]
    }
  },

  "permissions": "approve-all"
}

示例 2 空闲 Agent（Room 成员）

runtimeClass "gemini" 解析后，启动后等待 Room 中其他 agent 的审查请求。 

{
  "oarVersion": "0.1.0",

  "metadata": {
    "name": "code-reviewer"
  },

  "agentRoot": {
    "path": "workspace"
  },

  "acpAgent": {
    "systemPrompt": "你是代码审查专家，负责检查代码变更的正确性、安全性和风格。",
    "process": {
      "command": "gemini"
    }
  }
}

示例 3 最小配置

仅包含必需字段的最简配置。 

{
  "oarVersion": "0.1.0",
  "metadata": { "name": "quick-task" },
  "agentRoot": { "path": "workspace" },
  "acpAgent": {
    "process": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/claude-code-acp"]
    }
  }
}

acpAgent.session.*
